Description

We managed to create the game field and fill it with clues: now it's time to play! Let's give our users an opportunity to guess where the mines are with the help of our hints.

All numbers are still shown to the user, but the mines are not. To win, the user must find all the mines on the field by marking them.

Update the field input and add the coordinates grid as in the examples so that users can mark cells by entering their coordinates.

Objectives

Your upgraded program should meet the following requirements:

  1. After initializing the field, all numbers are shown to the user, but not the positions of the mines.
  2. The user sees the message Set/delete mines marks (x and y coordinates):  and enters two numbers as coordinates on the field.
  3. The user input is treated according to the rules:
    1. If the user enters the coordinates of a non-marked cell, the program marks the cell, which means that the user thinks a mine is located there.
    2. If the user enters the coordinates of a cell with a number, the program should print the message There is a number here!, ask the user again and not print the field, since cells with numbers are guaranteed to be free of mines.
    3. If the user enters the coordinates of a marked cell, the cell becomes unmarked. This is necessary because the game ends only if all marks are correct, but the user can mark more cells than there are mines.
  4. After successfully marking or unmarking a cell, the new field state is printed. The symbol . is used to represent non-marked cells and * is for the marked ones. The prompt for the user's next move is printed until the game is finished.
  5. When the user marks all mines correctly without marking any empty cells, they win and the game ends. If the user marked extra cells that do not contain mines, they continue playing, and after clearing all excess mine marks the user wins. Print the message Congratulations! You found all mines! after the final field state.

Examples

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.
